# Описание

«Шахматы» — классическая логическая игра для двух игроков, реализованная
как сетевое приложение.

Игроки управляют своими фигурами и поочередно делают ходы, следуя
правилам классических шахмат. Цель игры — поставить мат королю
соперника.

Игра должна поддерживать взаимодействие двух игроков в реальном времени
через клиент-серверную архитектуру с полной синхронизацией состояния
доски.

# Требования
## Сервер
- Управляет всей игровой логикой.
- Проверяет корректность ходов согласно правилам шахмат.
- Обновляет состояние доски.
- Отслеживает состояние партии (шах, мат, пат, ничья и др.).
- Обеспечивает очередность ходов.
- Отслеживает таймеры игроков (если используются).
- Синхронизирует состояние игры между клиентами. 

## Клиент
- Отображает шахматную доску и фигуры.
- Позволяет игроку выполнять ходы.
- Получает обновления состояния от сервера.
- Отображает текущий статус игры (ход, шах, результат и т.д.). 

# Протокол сообщений
Для клиентских обращений, сервер использует порт `TCP/8228` и собственный протокол обмена информацией.
## Структура
Протокол строится из двух частей:
- Первые биты, указывающие на тип пересылаемого сообщения (4 бит)
- Параметры, передаваемые в виде JSON строки.

## Типы сообщений и параметры

### Сообщения с клиента серверу (C2S)
В колонке "Код сообщения" перед 4 битами указывается буква `S`, обозначающая, что данное сообщение должно получаться и обрабатываться клиентом.

| Код сообщения | Условное наименование | Описание | Применение | Параметры |
|--|--|--|--|--|
| S0000 (0) | PING_REQUEST | Пинг сервера | Данный тип сообщений используется клиентом для проверки факта активности соединения с сервером. Ожидает получить в ответ `PING_RESPONSE (C0000)`. В случае, если сервер длительное время не получает данный запрос он запускает таймер ожидания игрока.| - 
| S0001 (1) | SERVER_CONNECTION | Подключение клиента к серверу | Отправляется на сервер в момент нажатия кнопки "Присоединиться к серверу" и подключает к соответствующей сессии. В ответ ожидает получить `CLIENT_CONNECTION (C0001)` от сервера. | `session_id` (int), `player_name` (string)
| S0010 (2) | PLAYER_UPDATE | Изменение статуса игрока | Используется для информирования сервера об изменении статуса игрока, в частности, его готовности к игре. После получения, сервер отправляет всем игрокам, кроме него самого, сообщение `OPPONENT_UPDATE (C0010)` для актуализации информации в интерфейсе. | `session_id` (int), `client_id` (int),`ready`(bool)
| S0011 (3) | - | - | - | -
| S0100 (4) | MOVE | Движение фигуры | Запрос, отправляемый серверу после подтверждения хода фигуры на клиенте. В случае корректности, всем клиентам придет ответный `GAME_UPDATE (C0100)` с новым состоянием доски, иначе - только на отправивший клиент `ILLEGAL_MOVE (C0101)` с предложением попробовать ещё раз. | `session_id` (int), `client_id` (int), `from` (byte), `to` (byte)
| S0101 (5) | RESIGN | Сдача | Сообщение, отправляемое после нажатия игроком на кнопку "Сдаться" в интерфейсе. После этого, на все клиенты отсылается `GAME_OVER (C0111)` с кодом `result: 2` | `session_id` (int), `client_id` (int)
| S0110 (6) | DRAW_OFFER | Предложение ничьей | Сообщение, отправляемое клиентом после нажатия в интерфейсе кнопки "Предложить ничью". После получения сообщения, сервер отсылает оппоненту `DRAW_DECISION (C0110)` с кодом `result: 0` | `session_id` (int), `client_id` (int)
| S0111 (7) | DRAW_RESPONSE | Ответ на предложение ничьей | Сообщение отправляемое с оппонента игрока, предложившего ничью. Распознается только, когда ранее пришло сообщение `DRAW_OFFER (C0110)` от другого игрока. | `session_id` (int), `client_id` (int), `agree` (bool) 
| S1000 (8) | DISCONNECTION | Отключение из игры | После нажатия в интерфейсе на кнопку "Отключиться от сервера" отсылается данный запрос, запускающий таймер, аналогичный тому, что запускается при отсутствии пинга. | `session_id` (int), `client_id` (int)
| S1001 (9) | SESSIONS_LIST_REQUEST | Список всех сессий | Запрос на получение списка всех активных сессий (комнат). Отсылает в ответ `SESSIONS_LIST_RESPONSE (C1001)` |
| S1010 (10) | CREATE_SESSION | Создание сессии | Запрос создаёт новую сессию и возвращает на клиент `CREATE_SESSION_RESPONSE (C1010)` | - 
| S1011 - S1111 (11-15) | - | - | - | - 

### Сообщения с сервера клиенту (S2C)
В колонке "Код сообщения" перед 4 битами указывается буква `C`, обозначающая, что данное сообщение должно получаться и обрабатываться клиентом.

| Код сообщения | Условное наименование | Описание | Применение | Параметры |
|--|--|--|--|--|
| С0000 (0) | PING_RESPONSE | Ответ на пинг от сервера | Данный тип сообщений используется сервером для ответа на входящий `PING_REQUEST (S0000)` с целью оповещения, что сервер всё ещё на связи. В случае, если длительное время не получает - отключает игрока от сервера с ошибкой `Timeout Error`. | -
| C0001 (1) | CLIENT_CONNECTION | Подключение сервера к клиенту | Ответное сообщение на `SERVER_CONNECTION (S0001)` от клиента. Сообщает клиенту об успешности подключения и выдаёт параметры (идентификатор и цвет) | `client_id` (int), `color` (int) 
| C0010 (2) | OPPONENT_UPDATE | Изменение статуса оппонента | Сообщение, отправляемое в ответ на `PLAYER_UPDATE (S0010)` второму игроку. Применяется для актуализации данных об оппоненте в интерфейсе. | `ready` (bool), `online` (bool)
| C0011 (3) | GAME_START | Изменение статуса начала игры | Применяется в процессе запуска партии. Создан для оповещения о том, что: <br><br> 1) Игра скоро начнется. <br><br> 2) Игра запущена. <br><br> 3) Таймер запуска отменен. | `time` (int)
| C0100 (4) | GAME_UPDATE | Изменение игры | Сообщение, отправляемое всем клиентам после успешно валидированного `MOVE (S0100)`. | `fen` (string), `check` (bool), `time_white` (long), `time_black` (long) 
| C0101 (5) | ILLEGAL_MOVE | Неразрешенный ход | Сообщение, присылаемое только приславшему крайний `MOVE (S0100)` в случае, если был совершен ход, не удовлетворяющий правилам игры. | `reason` (string) 
| C0110 (6) | DRAW_DECISION | Оповещение о решении соперника по ничьей | Используется для передачи запросов от двух игроков друг к другу. Подробнее читать в блоке с параметрами. | `type` (int) 
| C0111 (7) | GAME_OVER | Окончание игры | Отсылается всем подключенным клиентам, в случае, если игра была окончена по тем или иным причинам. Причина кодируется в `result` | `result` (int), `winner` (int) 
| C1000 (8) | PAUSE | Техническая пауза | Пауза, вызываемая в случае отключения/недоступности одного из клиентов. Длится ровно до того момента, пока игрок не переподключится, либо таймер не истечет. Во втором случае будет прислан `GAME_OVER (C0111)` с соответствующим `result`. | `status` (bool) 
| C1001 (9) | SESSIONS_LIST_RESPONSE | Ответ на список всех сессий | Возвращает массив из чисел, представляющих из себя набор структур `session` всех активных сессий. | `rooms` (array(session))
| C1010 (10) | CREATE_SESSION_RESPONSE | Ответ на создание сессии | Ответное сообщение, информирующее о новосозданной сессии | `session` (session)
| C1011 - C1110 (11-14) | - | - | - | - 
| C1111 (15) | EXCEPTION | Исключение | Оповещение клиента о любых ошибках, происходящих на сервере. | `code` (int), `text` (string) 

## Параметры

| Имя | Тип данных | Описание |
|--|--|--|
|session_id|`int`| Получается клиентом при подключении к серверу в ответном запросе `CLIENT_CONNECTION (C0001)`. Используется сервером для определения, к какой именно игровой сессии относится запрашиваемая команда. |
|client_id|`int`|Генерируемый сервером уникальный идентификатор присоединяющегося игрока. Используется для идентификации действий, совершаемых игроком. Получается вместе с `session_id` в `CLIENT_CONNECTION (C0001)`|

### SERVER_CONNECTION (S0001)


| Имя | Тип данных | Описание |
|--|--|--|
| player_name | `string` | Имя, указываемое игроком на клиенте непосредственно перед присоединением к серверу. Используется сервером для идентификации игрока при подключении. |

### PLAYER_UPDATE (S0010)
| Имя | Тип данных | Описание |
|--|--|--|
|ready| `bool` | Атрибут, указывающий на готовность игрока к запуску игры. |

### MOVE (S0100)
| Имя | Тип данных | Описание |
|--|--|--|
|from| `byte` | Указывает на расположение конкретной клетки, в которой сейчас находится фигура. |
|to| `byte` | Указывает на расположение конкретной клетки, в которую движется фигура. |

#### Кодирование позиции в числовое значение
1. Берётся `x` и `y` конекретной клетки. Например: `(6, 8)`.
2. Берем идентификаторы в массивах (отнимаем единицу): `(5, 7)`.
3. Берем двоичное представление обоих чисел: `101` и `111`. 
4. Конкатенируем и получаем новое число `101111`, которое можно представить в десятичной форме: `47`. Таким образом , получаем итоговое значение, которое записывается в сообщение.

### DRAW_RESPONSE (S0111)
| Имя | Тип данных | Описание |
|--|--|--|
|agree|`bool`|Отвечает за принятие предложение ничьей вторым игроком. В случае, если `agree` = `true` - всем клиентам отправляется `GAME_OVER (C0111)` с кодами `result: 1` и `winner: -1`, иначе - запросившему ничью `DRAW_DECISION (C0110)` с кодом `result: 1`|

### CLIENT_CONNECTION (C0001)
| Имя | Тип данных | Описание |
|--|--|--|
|color| `int` | Цвет фигур, выдаваемый игроку. `0` - белый, `1` - чёрный. |

### OPPONENT_UPDATE (C0010)
| Имя | Тип данных | Описание |
|--|--|--|
|ready|`bool`|Булевая переменная, в которой указывается факт готовности оппонента.|
|online|`bool`|Булевая переменная, в которой указывается факт того, что игрок в настоящий момент в сети.|

### GAME_START (C0011)
| Имя | Тип данных | Описание |
|--|--|--|
|time|`int`|Переменная, определяющая статус и таймер начала игры. Если `time > 0` - секунды таймера до начала игры; `time = 0` - игра запущена; `time = -1` - запуск игры отменен (Например, один из игроков отменил состояние готовности до истечения таймера).|

### GAME_UPDATE (C0100)
| Имя | Тип данных | Описание |
|--|--|--|
|fen|`string`|Расстановка элементов на доске в нотации FEN (Forsyth–Edwards Notation)|
|check|`bool`|Информация о том, поставлен ли шах|
|time_white|`long`|Время в миллисекундах, оставшееся у белых до истечения таймера. Указывается для синхронизации таймера сервера и локального клиента клиента.|
|time_black|`long`|Время в миллисекундах, оставшееся у чёрных до истечения таймера. Указывается для синхронизации таймера сервера и локального клиента клиента.|

### ILLEGAL_MOVE (C0101)
| Имя | Тип данных | Описание |
|--|--|--|
|reason|`string` | Строковое описание того, почему именно этот ход недопустим.|

### DRAW_DECISION (C0110)
| Имя | Тип данных | Описание |
|--|--|--|
|type|`int`|Тип пришедшего сообщения. `0` - Приходит игроку, когда оппонент запросил ничью и вызывает диалог принятия решения для него. `1` - Приходит игроку, отправившему запрос на ничью, когда оппонент отказался.|

### GAME_OVER (C0111)
| Имя | Тип данных | Описание |
|--|--|--|
|result|`int`|Причина, по которой игра окончилась. Ниже указываются значения параметра.|
|winner|`int`|Сторона, которая одержала победу. Ниже указываются значения параметра.|

#### Коды `result`
- `0` - мат
- `1` - ничья/пат
- `2` - сдача
- `3` - истчечение таймера
- `4` - игрок покинул игру

#### Коды `winner`
- `-1` - никто
- `0` - белые
- `1` - чёрныё


### PAUSE (C1000)
| Имя | Тип данных | Описание |
|--|--|--|
|status|`bool`| `true` - пауза запущена. `false` - пауза окончена. |

### EXCEPTION (C1111)
| Имя | Тип данных | Описание |
|--|--|--|
|code|`int`|Код сообщения|
|text|`string`|Описание происшествия|

#### Коды сообщений
| Код | Пояснение | Когда используется |
|--|--|--|
|0|Неизвестная ошибка|Всё остальное|
|1|Внутренняя ошибка сервера|Когда происходит баг в коде сервера|
|2|Ошибка формата параметров|Когда введенные параметры не соответствуют ожидаемым (Например, подается "text" вместо ожидаемого "client_id")|
|3|Ошибка типизации|Когда введенные параметры соответствуют ожидаемым, но не проходят по типам (Например, передано строковое значение вместо числового)|
|4|Не найдено|Когда созданный запрос создан от имени клиента, который не находится в данной игровой сессии|
|5|Ошибка правил игры|Когда указываются параметры, не соответствующие условиям игры (Например, указывается клетка за пределами 8x8)|
|6|Ошибка комнаты|Указывается, когда комната заполнена или недоступна (Например, данный игрок уже присоединен|

## Пример
- `5` (0101 - предложение сдачи).
- `{'player_side': 0}` - содержание, указывающее на то, какая именно сторона запросила сдачу.

Итоговое сообщение `5{'player_side': 0}`, отправляемое на сокет сервера TCP/8228

# Примерный сценарий

| Очередность | Хост | Описание |
|:---:|:---|:---|
| 1 | `КЛИЕНТ` | Игрок заходит в программу |
| 2 | `КЛИЕНТ` | Нажимает на кнопку "Создать комнату" -> Отправляется запрос на сервер `CREATE_SESSION (S1010)` |
| 3 | `СЕРВЕР` | Сервер отсылает запрос `CREATE_SESSION_RESPONSE (C1010)` обратно этому же клиенту. |
| 4 | `КЛИЕНТ` | Из пришедшего запроса вычленяется `session_id`, на основе которого формируется `SERVER_CONNECTION (S0001)` с прикреплённым `player_name`. |
| 5 | `СЕРВЕР` | Проверяет, не подключен ли уже этот игрок (по `player_name`). В случае неудачи отправляет `EXCEPTION (C1111)` с кодом `6`. В случае удачи — `CLIENT_CONNECTION (C0001)` со сгенерированным `client_id` и выданным `color`. |
| 6 | `КЛИЕНТ` | Запускается бесконечный цикл с задержкой `X` сек. Отсылается `PING_REQUEST (S0000)`, ожидается `PING_RESPONSE (C0000)`. При отсутствии ответа игрок кикается с `Timeout Error`. |
| 7 | `СЕРВЕР` | Запускается цикл с задержкой `2X` сек. Проверяет время последнего запроса от клиентов. При долгом отсутствии кикает игрока и шлёт оппоненту `OPPONENT_UPDATE (C0010)` с `online=false`. Запускает таймер тех. паузы `M` мин, по истечении засчитывает победу оставшемуся (информация через `PAUSE (C1000)`). При отключении обоих сессия закрывается. |
| 8 | `КЛИЕНТ` | После успешного подключения игрок попадает в лобби ожидания. За ним закрепляется идентификатор и цвет. |
| 9 | `КЛИЕНТ` | При нажатии "Готов" отсылается `PLAYER_UPDATE (S0010)` с `ready=true`. При отмене — тот же запрос с `ready=false`. |
| 10 | `СЕРВЕР` | Получая `PLAYER_UPDATE (S0010)`, отправляет оппоненту `OPPONENT_UPDATE (C0010)` для отображения в интерфейсе. |
| 11 | `СЕРВЕР` | При готовности всех запускает таймер, отправляя `GAME_START (C0011)` с `time=N`. По истечении шлёт `time=0` (запуск). Если до истечения пришёл `PLAYER_UPDATE (S0010)` с `ready=false`, шлёт `time=-1` (отмена). |
| 12 | `КЛИЕНТ` | При получении `time=N` запускает локальный таймер. При `time=0` моментально запускает игру. При `time=-1` уведомляет об остановке запуска. |
| 13 | `СЕРВЕР` | После запуска игры принимает запросы `(S0100-S0111)`, относящиеся к игровому процессу. |
| 14 | `СЕРВЕР` | Получая `MOVE (S0100)`, проверяет допустимость хода. В случае успеха отправляет `GAME_UPDATE (C0100)`, при неуспехе — `ILLEGAL_MOVE (C0101)`. |
| 15 | `КЛИЕНТ` | Получая данные о ходе, отображает актуальное поле (параметр `fen`), синхронизирует локальные таймеры и уведомляет о шахе. |
| 16 | `КЛИЕНТ` | Получая `ILLEGAL_MOVE (C0101)`, отображает сообщение о недопустимости хода и предлагает сделать его ещё раз. |
| 17 | `КЛИЕНТ` | При нажатии "Сдаться" отправляет на сервер `RESIGN (S0101)`. |
| 18 | `СЕРВЕР` | При получении `RESIGN (S0101)` отправляет всем `GAME_OVER (C0111)` с победой второго игрока и кодом `result=2`. |
| 19 | `КЛИЕНТ` | При нажатии "Предложить ничью" отправляет на сервер `DRAW_OFFER (S0110)`. |
| 20 | `СЕРВЕР` | При получении `DRAW_OFFER` отправляет второму клиенту `DRAW_DECISION (C0110)`. |
